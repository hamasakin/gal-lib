## 1. Tweaks Panel to All Pages (Phase 6 to 7/8/9/10)

Status: WIRED

## 5. End-to-End User Flows

### Flow A - Browse to Detail to Launch to Track Session: COMPLETE

1. / renders Library; GameGrid populated by searchGames (Library.tsx:86-97).
2. Click card; GameCard.tsx:131 navigate to /games/:id; Detail mounts.
3. Detail mounts; refreshGame + listSessions + listGameTags + getScreenshotSettings (lines 250-280).
4. Click LaunchButton; onLaunchClick (line 425); updateGameLaunchConfig then launchGame(gameId, true); backend emits active-session-changed.
5. main.tsx:78 listener writes session into store; ActiveSessionBar visible on Library, LaunchButton collapses to red square stop variant.
6. Game exits; backend emits null; store cleared; Detail prevActiveRef effect (line 311) re-fetches sessions; new row in session history.
7. Screenshot capture (per-game interval); on next visit to /screenshots or detail screenshots tab, getScreenshots(gameId) returns the new rows.

### Flow B - Settings to Add Root to Scan to Library Refresh: COMPLETE

1. /settings; scan-roots section; add-root; addScanRoot; refresh roots (Settings.tsx:149-166).
2. scan-ops section; full or incremental; startScan(mode); navigate to / (line 251).
3. Backend emits scan-progress; module-scope listener at main.tsx:45 updates store; ScanProgressBar re-renders.
4. On status equals completed, Library edge-detect effect (Library.tsx:117-135) refetches grid + sidebar and fires the rich toastScanFinished(total, ok, pending) toast.

### Flow C - Tweaks Live Reflow: COMPLETE

Toggling any axis writes to document.documentElement.dataset.* and repaints all CSS-variable consumers (grid columns, sidebar width, accent fills, radius, fonts).

### Flow D - Screenshots Page to Detail Cross-link: MOSTLY COMPLETE

Screenshots.tsx:156 view-game button calls navigate to /games/:id. Line 215 view-all routes to /games/:id?tab=screenshots, but Detail.tsx does not parse the tab query parameter; the destination renders with overview tab default. UX gap, not a blocker.

## 7. Findings

### Blockers
None.

### Warnings (visual or UX gaps; no integration breaks)

1. LIB-02 visual regression - HeroCard.tsx removed and GameGrid.tsx reverted to uniform auto-fill grid. The magazine asymmetric grid hero band 1.6fr+1+1+1 requirement and Phase 7 SUMMARY no longer match code. Header comment at GameGrid.tsx:8-13 explains the reason (portrait cover cropping + density mismatch). All other LIB-* reqs intact; only the hero band is missing. Affects: LIB-02.

2. TWK-03 / scan jump deferred with PGE-01 - TweaksPanel ships 5 jumps instead of 6. Scan jump intentionally absent because /scan route deferred. Internally consistent with Phase 9 deferral note. Affects: TWK-03.

3. UIPreferences theme row stale copy - src/components/settings/UIPreferences.tsx:135 still says theme switch will arrive in Phase 5. Phase 6 has shipped 3-theme switching via Tweaks panel. Disabled and harmless, but the copy misleads users. Cosmetic; not a wiring break.

4. Detail tab=screenshots deeplink ignored - Screenshots.tsx:215 builds /games/:id?tab=screenshots but Detail.tsx does not parse the tab query parameter. Tab defaults to overview. UX gap; flow still completes - user has to click the screenshots tab manually.

### Confirmed-OK Flows
- Library browse to detail open to launch to ActiveSessionBar appears to game exits to session row appears in detail (Flow A)
- Settings add-root to scan trigger to progress events update bar to completion toast + grid/sidebar refresh (Flow B)
- Tweaks panel mutations to live page repaint without re-render (Flow C)
- Screenshots page lightbox + cross-link to detail (Flow D, with one query-param caveat above)

## 8. Verdict

v1.1 cross-phase integration is READY TO SHIP with one acknowledged visual regression (LIB-02 hero band absent) and minor cosmetic copy debt. All five phases plug into the same Zustand stores, the same v1.0 IPC layer, and the same event subscriptions. No data flow is broken. PGE-01/PGE-02 are honestly marked as deferred with a clean v1.2 hand-off path.
