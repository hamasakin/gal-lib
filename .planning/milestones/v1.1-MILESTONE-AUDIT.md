## Unsatisfied Requirements

### LIB-02 — Magazine Asymmetric Grid (Phase 7) — REVERTED

**Spec:** 主区采用「杂志式不对称网格」——首行 hero band（1.6fr + 1 + 1 + 1 四联），后续行为 `repeat(auto-fill, minmax(--card-w, 1fr))` 等密度网格.

**Code state:** `src/components/library/GameGrid.tsx:8-13` documents the revert: portrait covers cropped poorly in the 1.6fr hero slot, and density toggle conflicted with the fixed hero ratio. `HeroCard.tsx` was removed. Grid is now uniform `auto-fill`. All other LIB-* requirements (cards, stamps, sidebar, page-hd, now-playing, paper-grain) are wired correctly.

**Resolution path:** Either (a) update REQUIREMENTS.md to drop the hero-band clause and accept the uniform grid as final, or (b) add a v1.2 phase to revisit hero band with the cropping/density issues solved.

### PGE-01 — Scan 进度页 KPI + 双栏 (Phase 9) — DEFERRED

**Spec:** Scan 进度页 KPI 4 联 + 双栏布局（feed + review queue）.

**Code state:** No `/scan` route. Scan UX lives entirely in `<ScanProgressBar>` on the Library page (Phase 7 restyled it). Phase 9 SUMMARY explicitly notes the deferral and lists prerequisites:
1. New router entry
2. Richer `scan-progress` event payload (per-directory rows, confidence, candidates)
3. Persistent review queue table (schema work)

**Resolution path:** v1.2 feature phase combining router + IPC + schema work.

### PGE-02 — 待复核卡片 + Bangumi/VNDB 候选 (Phase 9) — DEFERRED

**Spec:** 待复核卡片含游戏 mini-cover、Bangumi/VNDB 候选小卡片对比，可一键切换数据源.

**Code state:** Inline low-confidence handling via `MetadataPicker` at scan time (Library.tsx:302). Batch review queue not implemented — depends on PGE-01 backing infrastructure.

**Resolution path:** Bundled with PGE-01 in v1.2.
