# Quick 20260512d — Overlay followup + 前缀置信度修复

## 背景

Quick 20260512c 把 dialog overlay 加深后用户反馈"还是有些会溢出"——`alert-dialog.tsx` 用的是独立 Overlay 组件，没受 20260512c 的修改影响，仍然是 `bg-black/10`。

另一个反馈：`F:\galgame\アマエミDL版` 在 VNDB 上能搜出来但没自动绑。

### 根因

`title_clean::clean_title("アマエミDL版")` → `"アマエミ"` (`DL版` 被 RE_NOISE 干掉) ✅
VNDB search "アマエミ" 命中 `"アマエミ ～甘やかさせて♥もっとデキてる彼女～"` ✅

但 `match_score::score` 走 containment 分支：
- short = 4, long ≈ 18 → coverage ≈ 0.22
- 70 + 0.22 * 29 ≈ 76

76 < 80（AUTO_BIND_THRESHOLD）→ **不自动绑**。

短前缀在 galgame 命名里非常常见（用户用简称 / 砍掉 ～...～ 副标题 / 砍掉 ` -DL版`），需要给这类信号更高 baseline。

## 修复

### A. alert-dialog overlay 同步加深

`src/components/ui/alert-dialog.tsx:37`：
- `bg-black/10` → `bg-black/60`
- `backdrop-blur-xs` → `backdrop-blur-sm`

和 dialog.tsx 对齐。

### B. match_score：前缀包含 baseline 80

`src-tauri/src/metadata/match_score.rs:score`：

```rust
let is_prefix = (c.starts_with(&q) || q.starts_with(&c)) && short_len >= 3.0;
let baseline: u8 = if is_prefix { 80 } else { 70 };
let span = (99 - baseline) as f64;
return baseline + (coverage * span) as u8;
```

- 前缀且短边 ≥ 3 字符 → baseline 80 → 即使 coverage 很低也 ≥ 80
- 非前缀 containment 或 prefix 太短 → 保持 70 baseline（防止 "A" 命中 "ABCDEFG"）

### 新增测试

1. `prefix_containment_clears_auto_bind_threshold` — `アマエミ` vs `アマエミ ～甘やかさせて♥もっとデキてる彼女～` ≥ 80
2. `prefix_short_query_below_floor_stays_at_baseline_70` — `アマ` (2 chars) 不 promote
3. `non_prefix_containment_keeps_baseline_70` — substring 非前缀走 70 路径
