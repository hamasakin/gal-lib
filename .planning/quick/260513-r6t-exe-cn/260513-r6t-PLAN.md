# Quick Task 260513-r6t — PLAN

## 问题

1. **详情页"日区启动"按钮无效。** `src/routes/Detail.tsx:675` 把 `launchGame(gameId, false)` 写死了，`use_le` 永远是 false。`LaunchButton` 的 popover 列出的 4 个 profile（Japanese / Simplified Chinese / Traditional Chinese / Custom）都是 Locale Emulator profile，所以详情页这颗按钮按设计应该「永远走 LE」。主界面 `GameCard.tsx:478-480` 走的是 `onLaunch(true)` → `launchGame(game.id, true)`，可以正常起 LE。

2. **exe 自动匹配未优先 `_cn.exe` 后缀。** 中文化游戏通常会同时存在 `Game.exe` + `Game_cn.exe`（或 `Game_chs.exe`），当前 `score_exe`（`src-tauri/src/scan/exe_score.rs`）的打分逻辑里没有针对汉化后缀的偏好，扫描结果会随机命中两者中的一个。

3. **缺一个手动改 exe 路径的入口。** Detail 页其实已有 `已识别可执行文件` 文本框（行 1261-1271），但只能手敲路径，没有文件浏览器，UX 不闭环。

### Task 2 — exe 评分：`_cn.exe` 后缀加分

**Files:** `src-tauri/src/scan/exe_score.rs`

**Action:** 在 `score_exe` 里加一段中文化后缀打分：

```rust
// 汉化版偏好：_cn / _chs / _zh 后缀显著加分，
// 让 "Game_cn.exe" 在与 "Game.exe" 并存时胜出。
let cn_suffixes = ["_cn", "_chs", "_zh", "-cn", "-chs", "-zh"];
for suf in cn_suffixes.iter() {
    if name.ends_with(suf) {
        score += 20;
        break;
    }
}
```

放在 prefix-match（`+5`）之后即可。+20 的量级足够压过 prefix-match + size + name-len（最大也就 +8），不会被 bad-name penalty（-10）救回。同时把模块顶部的 doc-comment 列表里新增一行说明，保持「契约即注释」。

**Verify:**
- `cargo test -p gal-lib-tauri scan::exe_score`（或 `cargo test` 整跑）通过；
- 新增单测 `prefers_cn_suffix_over_plain`：同目录下 `Game.exe`（2MB）和 `Game_cn.exe`（2MB），后者得分严格更高。

**Done:** 含 `_cn.exe` / `_chs.exe` / `_zh.exe` 的目录，扫描自动选中带后缀的版本。
