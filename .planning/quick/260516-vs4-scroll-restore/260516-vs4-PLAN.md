<objective>
从详情页返回游戏库列表页时，恢复进入详情页之前的滚动条位置。

Purpose: 当前 Library 和 Detail 是 HashRouter 的兄弟路由，导航到详情页时 Library 整个组件卸载，组件内部状态（含滚动位置）全部丢失，返回时 `overflow-auto` 容器的 scrollTop 重置为 0。用户在大型游戏库里翻到中段点开一个游戏看完返回，又得从头滚一遍，体验割裂。

Output: 滚动位置存入 Zustand `useLibraryStore`（跨卸载存活），Library 卸载前快照、重新挂载后恢复。
</objective>

<!-- 关键事实，执行时无需再翻代码库 -->

路由结构（src/router.tsx）：HashRouter，`<App/>` 是 layout，`<Library/>`（index `/`）与 `<Detail/>`（`/games/:id`）是 <Outlet/> 下的兄弟路由 —— 导航到详情页时 Library 完全卸载，组件内 useState/useRef 全部丢失。

滚动容器（src/routes/Library.tsx 第 570-573 行）：
```tsx
<div
  ref={scrollContainerRef}
  className="flex min-h-0 flex-1 flex-col overflow-auto"
>
```
`scrollContainerRef` 是 `useRef<HTMLDivElement>(null)`（第 208 行）。这是唯一的滚动区，网格视图和列表视图都滚这一层。

虚拟化：网格视图 GameGrid 用 `@tanstack/react-virtual` row-mode，`getScrollElement: () => scrollContainerRef.current`。virtualizer 读原生 `scrollTop` 派发的 scroll 事件来更新 virtualItems —— 因此直接写 `el.scrollTop` 即可恢复，无需虚拟化库的特殊 API。列表视图 GameList 是普通 DOM，不虚拟化。

恢复时机关键点：网格视图首帧的 `getTotalSize()` 依赖 ResizeObserver 测出 columnCount/cardWidth，首次 render 时 `scrollHeight` 可能还不够大，过早写 scrollTop 会被浏览器夹回。恢复 effect 需放在 `games.length > 0` 且让一帧布局完成后再写（用 requestAnimationFrame 或两层 rAF）。

useSmoothWheel（src/hooks/useSmoothWheel.ts）：它的 `target` 在 effect 内用 `let target = el.scrollTop` 惰性初始化，且仅在首次 wheel 事件时若 `raf == null` 才重新对齐到 `el.scrollTop`。因此恢复 scrollTop 不会和 smooth wheel 打架 —— 只要恢复发生在用户首次滚轮之前即可，无需改 useSmoothWheel。

<task type="auto">
  <name>Task 2: Library 卸载前快照、挂载后恢复 scrollTop</name>
  <files>src/routes/Library.tsx</files>
  <action>
在 `Library()` 组件内：

1. 从 store 取 setter：`const setLibraryScrollTop = useLibraryStore((s) => s.setLibraryScrollTop);`。
   滚动位置的读取不要用响应式订阅（会触发无谓 re-render），在恢复 effect 内用 `useLibraryStore.getState().libraryScrollTop` 一次性读取。

2. 卸载前快照 —— 新增一个 effect，依赖数组为空 `[]`，return 一个 cleanup：cleanup 内读 `scrollContainerRef.current?.scrollTop`，若存在则调用 `setLibraryScrollTop(...)`。这样组件卸载（导航到 Detail）时把当前 scrollTop 写进 store。
   说明注释：cleanup 而非 scroll 事件监听，避免高频滚动时每帧写 store；只在离开页面那一刻取一次值。

3. 挂载后恢复 —— 新增一个 effect，依赖数组为空 `[]`，在内部判断：读 `const saved = useLibraryStore.getState().libraryScrollTop;`，若 `saved <= 0` 直接 return（首次进入保持顶部）。否则用嵌套两层 `requestAnimationFrame` 把 `scrollContainerRef.current.scrollTop = saved` 延后到布局稳定后执行 —— 第一帧让 GameGrid 的 ResizeObserver 测出 columnCount、virtualizer 算出 totalHeight 撑起 scrollHeight，第二帧再写 scrollTop，否则 scrollTop 会被不足的 scrollHeight 夹回。effect 内保存 rAF id，cleanup 里 `cancelAnimationFrame` 两个 id（防止卸载后回调仍执行）。
   说明注释：解释虚拟列表场景为何需要延后到 totalHeight 撑起后再写；并说明 useSmoothWheel 的 target 惰性初始化 + 首 wheel 才重对齐，所以恢复 scrollTop 不会与平滑滚动冲突。

不要修改 GameGrid / GameList / useSmoothWheel。恢复 effect 放在已有的 `scrollContainerRef` 声明（第 208 行）和 `useSmoothWheel` 调用（第 213 行）之后即可。
  </action>
