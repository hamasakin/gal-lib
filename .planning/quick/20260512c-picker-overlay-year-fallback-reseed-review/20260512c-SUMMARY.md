## Bug B — Bangumi 年份 infobox fallback

Bangumi 顶层 `date` 字段在很多 game subjects 是空字符串（doujin / 老游戏 / 未发售），导致 `parse_release_year("")` → None → 年份字段保持 NULL。

- `src-tauri/src/metadata/bangumi.rs`：
  - 新增 `extract_release_date_from_infobox()`：扫描 `发售日 / 发行日 / 発売日 / 发行日期 / 发售日期` keys，取 string value
  - 新增 `normalize_jp_date()`：`2020年4月24日` / `2020年4月` / `2020年` → `2020-04-24` / `2020-04` / `2020`
  - `fetch_detail()` 兜底逻辑：`raw.date.filter(|s| !s.trim().is_empty()).or_else(|| extract_release_date_from_infobox(...))`
- 5 个新单测覆盖 ISO / 日式 / 年-月 / 仅年 / 缺失场景

下游 `parse_release_year` 拿到 `2020-XX-XX` 前 4 位自然解析为 2020；下游 5 处 UPDATE 路径已经在 quick 20260512b 中接好。**已绑定 Bangumi 但年份空的老游戏**：需要用户跑"全部刷新元数据"或 backfill 触发回填。

## Bug C — 复核队列回灌（reseed）

历史 `metadata_source='none'` 游戏从未经过 `sync_review_queue_for_game`（Phase 12 才加），所以不在 `scan_review_queue` 表里，Scan 页看不到。

- `src-tauri/src/commands.rs`：新增 `reseed_review_queue() -> i64` IPC：
  - 一条 SQL `INSERT OR REPLACE` 把 `metadata_source='none'` OR (`!='manual'` AND `confidence<80`) 的游戏全灌入队列
  - 排除 manual 绑定（用户已确认）
  - 返回插入/替换行数
- `src-tauri/src/lib.rs`：注册到 `invoke_handler!`
- `src/lib/scanReview.ts`：`reseedReviewQueue()` TS 包装
- `src/routes/Scan.tsx`：顶栏新按钮 `重新生成待复核队列`（ListRestart icon）+ `reseedSeq` state，调用后 bump seq → toast 显示灌入数 → 刷新 KPI
- `src/components/library/ReviewQueue.tsx`：接收新 prop `reseedSeq?: number`，在 effect 里依赖它触发 refetch
