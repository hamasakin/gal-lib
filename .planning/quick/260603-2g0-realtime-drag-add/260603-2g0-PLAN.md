# Quick 260603-2g0 — 拖曳目录添加游戏实时刷新

## 问题根因

`src/App.tsx` 的拖拽 drop 处理用 `for (p of paths) await addGame(p)` 串行等每个游戏
**完整跑完元数据抓取**（Bangumi/VNDB 网络请求）后才进下一个，且**只在整批结束后**才
`searchGames` 刷新一次列表 → 拖入后长时间无反应。

而 `add_game` 命令不接 `AppHandle`、不发任何事件，没接上项目里现成的实时管线：
`start_scan` 每插入一个占位行 `emit("games-changed")`，`Library.tsx` 订阅 `games-changed`
节流刷新网格（占位卡先出现 → 元数据回填）。

## 方案（用户已确认：全部占位卡瞬间出现）

新增批量后端命令 `add_games(paths)`，复用现成两阶段 ingest：
- **Phase 1**：遍历所有路径 → 校验 + `insert_placeholder_dir` → 每条 `emit("games-changed")`
  （所有占位卡秒出，带「获取中」徽标）。收集成功的 `(game_id, dg)`，统计失败数。
- **Phase 2**：逐个 enrich → `meta-fetch-progress started` → `enrich_metadata_for_dir`
  → `games-changed` + `meta-fetch-progress finished`（卡片逐个回填元数据 + 脉冲高亮）。
- 返回 `{ added, failed }`。

前端 `App.tsx` 拖拽循环改为单次 `await addGames(paths)`，落点立刻弹 loading toast
（动态反馈），结束后转 success/partial。Library 的 `games-changed` 订阅负责实时网格刷新；
保留结束后的兜底 `searchGames` 刷新（覆盖落点时不在 Library 路由的情况）。

## 任务

1. **Rust**：`src-tauri/src/commands.rs` 新增 `add_games` 命令；`src-tauri/src/lib.rs` 注册。
2. **前端 lib**：`src/lib/scan.ts` 加 `addGames(paths)` invoke 包装 + 返回类型。
3. **前端 App**：`src/App.tsx` drop 循环改用 `addGames` + loading toast。
4. **i18n**：三语加 `dragdrop.adding` loading 文案。
