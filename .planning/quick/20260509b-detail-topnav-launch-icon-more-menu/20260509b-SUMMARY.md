# Quick Task Summary: detail-topnav-launch-icon-more-menu

## What shipped

### 1. Top nav bar on the Detail route (commit 53a9182)

A new sticky 52px header above the hero with three zones:

```
┌──────────────────────────────────────────────────────────────────┐
│ ← 返回图书馆 [Esc]   图书馆 / 雪月华社 / 向日葵盛开的盛夏     ← 12/16 → │
└──────────────────────────────────────────────────────────────────┘
```

- **Back button** — clickable pill with an `Esc` keyboard hint badge.
  Window-level keydown listener captures Escape and calls
  `navigate(-1)`. Radix Dialog primitives (used by every modal in the
  app — AlertDialog, MetadataPicker, Lightbox) trap focus and call
  stopPropagation on their own Esc handlers, so the global listener
  only fires when no modal is open.
- **Breadcrumb** — three segments. 图书馆 navigates to `/`. The
  brand segment sets `store.filter = { brand: game.brand }` then
  navigates `/`. The title segment is the current page (no link),
  truncates with ellipsis at 280px. Brand segment is hidden when
  `game.brand` is null.
- **Prev/next + N/M counter** — driven by the (currently filtered/
  sorted) `useLibraryStore.games` array. When the user deep-linked
  to a game id that isn't in the list (Library hasn't hydrated yet),
  `idx === -1`, arrows disable, and the counter renders an em-dash.

The previous inline back-button + brand/year mini-row that sat at the
top of the hero info column has been removed — it was a partial
breadcrumb that linked nothing and duplicated information now in the
real breadcrumb.

### 2. Launch-button play-icon centering (commit 5d49445)

The right-pointing Play triangle has its visual centroid offset to the
left of its geometric center (the wide base is on the left side,
the apex on the right is a single point). Centered geometrically, it
*looks* off-center to the left. A small `translateX` corrects that:

- `LaunchButton.tsx` — Play icon `translateX(1.5px)` for the 16px
  icon in the 44px collapsed-state circle.
- `GameCard.tsx` hover overlay — `translateX(1px)` for the smaller
  14px icon in the 30px hover-overlay button (smaller icon → smaller
  offset).

Square (active-state stop icon) is untouched because it's symmetric.

### 3. 更多 dropdown menu (commit 53a9182)

The previously inert MoreHorizontal button next to the launch button
becomes a shadcn `DropdownMenu` with two entries:

- **打开本地目录** — invokes a new Tauri command `open_in_explorer`
  that runs `explorer.exe {game.path}` after a `Path::exists()`
  guard. Stale `games.path` (deleted directory after scan) surfaces
  as `路径不存在：{path}` toast instead of Explorer's generic
  "location not available" dialog.
- **复制路径** — same handler the right-sidebar 路径 section's 复制
  路径 button uses. Kept duplicate by design: dropdown is the hot
  path, sidebar is for path inspection.

`open_in_explorer` is the 44th Tauri command (43 → 44 since 05b's
count). No security concern: `Command::arg(path)` passes the path as
a separate argv entry, so there's no shell-interpretation / command
injection vector. The path itself originates from `games.path` which
the user added during scan.

## Decisions

- **Top bar uses `sticky top-0 z-30`.** The outer Detail wrapper is
  the scroll context (`relative h-full w-full overflow-auto`), so
  sticky resolves against that container correctly.
- **Esc handler at window level**, not on the route div. Window-level
  works regardless of focus. Radix Dialogs intercept their own Esc
  before it bubbles, so global handler only fires when no modal is
  open — confirmed by reading shadcn dialog primitives earlier in the
  session.
- **Brand breadcrumb-click behaviour: replace `filter` slice
  entirely.** Mirrors the Sidebar's single-axis activation rule
  documented in the 04d trend. The user's existing tag/status/
  favorite/decade filters get cleared, which is what they expect when
  they click "show me more from this brand".
- **Prev/next siblings sourced from `store.games`.** Already
  filtered/sorted by Library.tsx — so the navigation respects the
  user's current view. Deep-linked entries get blank counter +
  disabled arrows (acceptable; same UX as before since there was no
  list to navigate at all).
- **Play-icon offset values (1.5px / 1px)** chosen by visual
  inspection of the rendered icons in dev mode (in a previous
  session); kept in this PR to match the consistent value across the
  two Play sites.
- **Backend command location: `commands.rs`** (one new function
  appended at end). Project convention puts every Tauri command
  handler in commands.rs.
- **Path validation in backend, not frontend.** The deleted-directory
  case is the only realistic error mode and has a Chinese-localised
  error string; the frontend just toasts `String(e)`.
