# Quick Task 20260512 — 封面 cache-buster

## 问题

`bind_metadata` / `refresh_metadata` 都把新封面写到磁盘上的同一个文件（`data/covers/{game_id}.{ext}`），但 `games.cover_path` 字段也是同一个字符串。前端：

```ts
convertFileSrc(`${dataDir}/${game.cover_path}`)
```

生成的 `<img src>` 在元数据更新前后**完全相同**。WebView image cache 按 URL 匹配，命中旧图 → 用户看到的封面不变。

## 修复

在 4 处 `convertFileSrc(...)` 调用后面追加 `?v=${last_scanned_at}` 作为 cache-buster。`last_scanned_at` 在 `bind_metadata` 和 `refresh_metadata` 的 SQL 里都会写 `datetime('now')`，每次刷新就变化一次。

## 改动

| 文件 | 行 | 改动 |
|---|---|---|
| `src/routes/Detail.tsx` | 578 | `convertFileSrc(...) + ?v=...` |
| `src/components/library/GameGrid.tsx` | 84 | `convertFileSrc(...) + ?v=...` |
| `src/components/library/ActiveSessionBar.tsx` | 84 | `convertFileSrc(...) + ?v=...` |
| `src/routes/Stats.tsx` | 660、667 | Pick 类型加 `last_scanned_at` + `convertFileSrc(...) + ?v=...` |

格式统一为：
```ts
convertFileSrc(`${dataDir.replace(/\\/g, "/")}/${game.cover_path}`) +
  `?v=${encodeURIComponent(game.last_scanned_at ?? "")}`
```

`last_scanned_at` 是 ISO 时间戳（含空格冒号），用 `encodeURIComponent` 包一下。
若为 NULL（理论不该发生，因为 cover_path 非 NULL 时它必非 NULL），退化为空串，行为同改前。

## 验证

- `pnpm tsc --noEmit` 通过
- 手动：bind_metadata 后 Detail/Library/ActiveSessionBar 立即显示新封面（无需刷新页面）

## 不做

- 不动后端：`cover_cache.rs` 写文件名继续用 `{game_id}.{ext}` —— 文件名稳定才能让 `cover_path` 字段也稳定，前端兜底有效
- 不加 service-worker / 全局 fetch interceptor
- 不分开"刷封面"和"刷元数据"语义（之前讨论的 A/B/C 方向另议）
