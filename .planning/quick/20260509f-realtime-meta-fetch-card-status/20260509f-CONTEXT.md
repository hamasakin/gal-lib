# Quick Task 20260509f: 扫描时实时给"正在获取元数据"的卡片加状态样式 — Context

**Gathered:** 2026-05-09
**Status:** Ready for planning

<domain>
## Task Boundary

让 GameCard 在元数据正在获取时实时显示一个高亮样式（脉冲边框 + spinner 图标），覆盖 4 个场景：
1. `start_scan` 全量首次扫描（新游戏）
2. `start_scan` 增量扫描（已有卡片再次刷新）
3. `refresh_all_metadata` 全量刷新元数据（所有卡片轮询）
4. `refresh_metadata` / `bind_metadata` 单条手动重抓

不包含：封面下载进度细分、单卡的百分比进度、整张卡黑蒙层覆盖、用户右键操作时的临时反馈（与现有 toast 重叠不必要）。

</domain>

<decisions>
## Implementation Decisions

### 视觉效果
- **脉冲边框 + spinner**：cover 外圈 2px `var(--accent)` ring + 呼吸 pulse 动画；现有 `获取中` badge 内嵌 `Loader2 className="animate-spin"` 图标
- 该样式优先级最高（覆盖 `待复核`、`无 EXE` badge）
- 不用整张卡黑蒙层（视觉太重）、不用底部 shimmer 进度条（与顶部 ScanProgressBar 风格冲突）
- 不拆后端为多阶段进度事件（搜索/下封面/写入），避免实现量爆炸

### 后端事件 schema（新事件）
- 新事件 `meta-fetch-progress`，独立于 `scan-progress`
- payload：`{ game_id: i64, phase: "started" | "finished" }`
- 前端订阅维护一个 `fetchingMetaIds: Set<number>` zustand 切片

### 首次扫描的占位行
- **后端 `start_scan` discover 完成后立即批量插入占位 `games` 行**（最小字段：path / raw_name / clean_name / executable_path；`metadata_source=NULL` `last_scanned_at=NULL`，沿用现有 `pending` 语义），再 `emit games-changed` / 让前端 refetch
- 然后 ingest 循环逐个 `emit meta-fetch-progress { phase: "started" }` → enrich 元数据 → `emit meta-fetch-progress { phase: "finished" }`
- 取消扫描后占位行**保留**（用户可右键手动重抓，行为对齐现有 `add_game`）
- **启动时清理孤儿占位行**：在 `lib.rs` 的 setup 路径或 schema 迁移后跑一次 `DELETE FROM games WHERE metadata_source IS NULL AND last_scanned_at IS NULL`，防止扫描中途崩溃留下半成品

### 拆分 `ingest_one_dir`
- 当前 `ingest_one_dir(pool, data_dir, dg) -> Result<i64, ...>` 一次性完成 INSERT + 元数据搜索 + 封面下载 + UPDATE
- 拆为两阶段：
  - `insert_placeholder(pool, dg) -> Result<i64, ...>`：仅 INSERT 最小行
  - `enrich_metadata(pool, data_dir, game_id, dg) -> Result<(), ...>`：跑元数据 + 下封面 + UPDATE
- `add_game` 命令保持串联调用 `insert_placeholder` + `enrich_metadata` 行为不变
- `refresh_all_metadata` / `refresh_metadata` / `bind_metadata` 也在边缘 emit `started` / `finished`

### 前端 store + GameCard
- `src/store/library.ts` 新增 `fetchingMetaIds: Record<number, true>` 切片 + setters：`addFetchingMetaId(id)` / `removeFetchingMetaId(id)` / `clearFetchingMetaIds()`
- `scan-progress` 收到 terminal status (`completed` / `cancelled` / `failed`) 时调用 `clearFetchingMetaIds()` 兜底
- `src/lib/scan.ts` 新 type + `onMetaFetchProgress` listener
- `src/main.tsx` 模块级订阅，按 `phase` 字段 add / remove
- `src/components/library/GameCard.tsx` 从 store 读 `isFetchingMeta = !!fetchingMetaIds[game.id]`：
  - 当 `true`：cover wrapper 加 `class="ring-2 ring-[var(--accent)] pulse-ring"`，bottom-left badge 强制为 `获取中` 并嵌入 `<Loader2 size={9} className="mr-1 animate-spin" />`
  - 当 `false`：保留现有 pending/review/no-exe 静态判断逻辑

### CSS 动画
- `src/index.css` 新增 `@keyframes pulse-ring` 关键帧（opacity 1 → 0.5 → 1，1.6s 循环）+ `.pulse-ring` 类
- 复用现有 `--accent` 色 token

### Claude's Discretion
- 占位行 INSERT 的具体 SQL 形态（直接 INSERT 还是 UPSERT，哪些字段必填、是否设 `name = raw_name`）
- 事件订阅在 `main.tsx` 还是新建独立模块文件
- spinner 图标尺寸/位置微调（badge 内嵌 vs cover 中央 floating）
- pulse-ring 关键帧具体节奏（呼吸频率、最低 opacity）
- 启动时清理逻辑放在 `lib.rs::setup` 还是 `db.rs::run_migrations` 后

</decisions>
