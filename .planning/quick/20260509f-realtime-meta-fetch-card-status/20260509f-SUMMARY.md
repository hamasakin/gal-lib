# Quick Task 20260509f: 元数据抓取实时高亮 — Summary

## One-liner

扫描 / 刷新元数据时，后端通过 `meta-fetch-progress { game_id, phase }` 事件实时通知前端正在 fetch 的卡片，GameCard 立即叠加 `pulse-ring` 边框 + Loader2 spinner 「获取中」 badge；扫描终态由 scan-progress 兜底 clearFetchingMetaIds，崩溃残留的孤儿占位行在 setup 路径自动清理。

## What Changed

### 后端 (Task 1 — commit 127a524)

**ingest 拆分两阶段（commands.rs）**

| Helper | Lines | 职责 |
|--------|-------|------|
| `insert_placeholder_dir` | commands.rs:76-109 | 幂等占位 INSERT（path / name / executable_path / screenshot_interval_sec=0），返回 game_id。第二次调用走 ON CONFLICT(path) 路径返回已有 id。 |
| `enrich_metadata_for_dir` | commands.rs:115-143 | 跑 `ingest::process_game` + UPDATE games 行（含 last_scanned_at=datetime('now')）。 |
| `ingest_one_dir` | commands.rs:155-163 | 旧入口保留：`insert_placeholder_dir` + `enrich_metadata_for_dir` 串联，供 `add_game` 调用。 |

`ingest.rs` 仅改顶部模块文档（line 22-34）说明两阶段契约，函数签名 0 改动。

**4 处 emit meta-fetch-progress**

| 位置 | 行号 | 错误路径处理 |
|------|------|-------------|
| start_scan ingest 循环 | commands.rs:423-431 (started/finished 紧贴 `enrich_metadata_for_dir`) | enrich 内 `let _ =` 吞错，循环继续；scan-progress completed 兜底清除 |
| refresh_metadata | commands.rs:661-664 (started) / commands.rs:714-717 (finished) | 中间逻辑包在 `let result: Result<(), String> = async { ... }.await;` —— 不论 Ok/Err，finished emit 永远跑，再 return result |
| bind_metadata | commands.rs:581-584 (started) / commands.rs:642-645 (finished) | 同样 async block 包裹模式 |
| refresh_all_metadata | commands.rs:798-801 (started) / commands.rs:838-841 (finished) | spawn 内 per-row 循环；UPDATE 失败被 `let _ =` 吞掉但 finished 永远 emit；末尾 scan-progress completed 兜底 |

`add_game` 路径**没有** emit — 单条主动添加场景前端不会先看到占位卡再看到 enrich 完成的中间态（用户点完按钮卡片才出现，那时 ingest 已结束）。

**孤儿占位清理（lib.rs:160-185）**

放在 setup 闭包内、`main_window.on_window_event(...)`（close-to-tray handler）之后，`Ok(())` return 之前。用 `tauri::async_runtime::spawn` 拉到异步上下文跑 `DELETE FROM games WHERE metadata_source IS NULL AND last_scanned_at IS NULL`。失败非致命（best-effort）。决策依据：本进程自己的 ingest 循环还没启动，正在跑的扫描属于上一进程（已死），不会误删活跃数据。
