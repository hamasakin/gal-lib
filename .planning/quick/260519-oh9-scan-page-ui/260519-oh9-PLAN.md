  <name>Task 1: ReviewQueue 加 max-height 让列表块内滚动</name>
  <files>src/components/library/ReviewQueue.tsx</files>
  <action>
给「待复核」列表一个有界高度上限，使其内部已有的 `overflow-y-auto` 真正接管滚动 —— 条目超出后在 ReviewQueue 块内部滚动，不再把整个 `/scan` 页撑长。

实现要点：
- 当前 ReviewQueue 根 div（约 209 行）是 `flex h-full min-h-[420px] flex-col`，内部列表区（约 221 行）是 `<div className="flex-1 overflow-y-auto">`。问题根因：`/scan` 页根容器自身是 `overflow-y-auto` 整页滚动容器，且未给 ReviewQueue 高度上限，导致 `flex-1` 列表区被无限撑高、滚动落到整页。
- 在根 div 的 className 中**追加**一个 `max-h-[...]` 约束（`h-full min-h-[420px]` 全部保留，仅追加 max-h）。这样根 div 拿到有界高度后，其内部 `flex-1 overflow-y-auto` 列表区自然在块内滚动。
- max-height 取值与左栏 ScanFeed 视觉协调：ScanFeed 没有显式 max-h，其可视高度受视口约束。采用基于视口的值 `max-h-[calc(100vh-280px)]`（280px 预留给 sticky 进度条 + PageHeader + KPI 条 + 上下 padding 的大致高度）—— 这样两栏整体高度上限一致、随窗口高度自适应。
- 不要修改 ReviewQueue 任何数据逻辑、事件订阅（scan-progress / meta-fetch-progress / reseedSeq）、采用/不再提示/手工绑定逻辑、MetadataPicker 渲染。
- 不要修改 ScanFeed.tsx。
