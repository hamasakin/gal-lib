# Quick 20260512b — Summary

修了 3 个用户反馈 bug：发行年份字段空 / 制作团队 chip hover 文字看不见 / 网格密度切换失效。

## 改动

### Bug 1 — 发行年份不再为空

后端 5 处 UPDATE 写入路径都没把 `release_year` 落库，虽然 schema v4 早就有这列。

- `src-tauri/src/ingest.rs`
  - 新增 `pub fn parse_release_year(s: Option<&str>) -> Option<i32>` — 从 `YYYY-MM-DD`/`YYYY-MM`/`YYYY` 解析前 4 位年份，限制 `1980..=2100`
  - `IngestResult` 加 `release_year: Option<i32>` 字段
  - `fetch_enrichment` 返回 tuple 多带 `Option<i32>`（取 `detail.release_date` 解析）
  - `process_game` / `process_game_cached` 把 release_year 设进 result
- `src-tauri/src/commands.rs` — 5 处 UPDATE 全部加 `release_year = COALESCE(?, release_year)`：
  - `apply_ingest_result`（scan / add_game）
  - `bind_metadata`（现场从 `detail.release_date` 解析）
  - `refresh_metadata`
  - `refresh_all_metadata`
  - `backfill_metadata_enrichment`（现场解析 + synthetic IngestResult 加 release_year）
  - 还修了 `bind_metadata` / backfill 里两处 synthetic `IngestResult` 缺字段

COALESCE 保护手动设置的 release_year 不被源 NULL 覆盖（和 brand / age_rating 对称）。

### Bug 2 — chip hover 文字不再消失

`tailwind.config.ts` 把 `accent` 映射到 shadcn 的 surface elevated 颜色 (`var(--bg-2)`)，brand 强调色才是 `brand.DEFAULT`。组件写 `hover:text-accent` 时文字色会等同于 `bg-2`，正好和 `hover:bg-bg-2` 同色 → 不可见。

- `src/routes/Detail.tsx`
  - L1516（ExternalLinkChip）：`hover:text-accent` → `hover:text-brand`
  - L1553（PersonChip）：同上

shadcn 内部用的 `bg-accent` + `text-accent-foreground` 配对正确（surface + 对应前景），不动。

### Bug 3 — 密度切换实时生效

`GameGrid.tsx` 的 measure 只在 mount 时跑 + 装 ResizeObserver。切换密度只改 CSS 变量 `--card-w`，容器 `clientWidth` 不变，RO 不触发。

- `src/components/library/GameGrid.tsx`
  - import `usePreferencesStore`
  - 订阅 `density`，加进 `useEffect` 的 deps → 密度变化时 effect 重跑，重新读 `--card-w` 并更新 `cardWidth` / `columnCount`
