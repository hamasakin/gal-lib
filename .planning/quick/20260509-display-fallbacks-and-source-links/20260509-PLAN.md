# Quick Task: display-fallbacks-and-source-links

## Goal

Three small frontend changes that improve the "未匹配元数据" experience and let
the user pivot from a game's detail page directly into the data-source web
sites for follow-up research:

1. **Basename fallback for unmatched games.** When `metadata_source` is null
   or `"none"`, game-name display falls back to the raw directory basename
   (the last segment of `game.path`) instead of the cleaned `game.name` —
   which has had paren content / noise prefixes / version trails stripped
   and can become an unrecognisable stub.
2. **Detail-page external links.** When `bangumi_id` / `vndb_id` is set,
   the corresponding pill in the hero and the corresponding row in the
   right sidebar's `条目信息` `<dl>` open the upstream entry page on
   click. A new `搜索源` section is added under the existing `路径`
   section with two buttons that search the data sources by `game.name`.
3. **Picker title fallback.** The `MetadataPicker` dialog title
   ("重新匹配元数据 — {name}") uses the same display fallback so the
   "match this directory" flow shows something the user can recognise.

## Why

- Currently when ingest fails to match a game, the cleaned title (e.g.
  `clean_title` strips paren-suffixes and version trails) leaves a stub
  that doesn't match what the user sees on disk. Showing the directory
  basename makes the row identifiable so the user can re-match
  manually.
- The detail page already shows `BGM · {id}` and `VNDB` rows but they
  aren't actionable. Letting them open Bangumi / VNDB directly removes
  a copy-paste step. Adding name-based search buttons gives a path even
  when there's no bound id.

## Out of scope

- Backend changes to the ingest pipeline. The current behaviour
  (`clean_title` storing the cleaned name in `games.name`) stays — the
  fallback is purely display-side so it works retroactively for games
  already in the DB.
- Adding `tauri-plugin-shell` / `tauri-plugin-opener`. Reuse the
  existing `window.open(url, '_blank')` pattern (already present in
  `Detail.tsx` for cover_url). If Tauri 2 default behaviour later
  intercepts these in-webview, a follow-up quick task can swap to a
  proper opener.
- Search affordances in components other than the Detail page (e.g.
  GameCard hover, sidebar). Keep the surface narrow — the user asked
  specifically for detail-page jumps.

## Tasks

### Task 1 — `src/lib/display.ts` helpers

**Files:** `src/lib/display.ts` (new).

**Action:**
- Export `basenameFromPath(path: string): string` — handles both `/` and
  `\\` separators (Windows + POSIX); returns the last non-empty segment
  or `path` itself if no separator found.
- Export `displayGameName(game: Pick<Game, "name" | "name_cn" | "metadata_source" | "path">): string`:
  - if `name_cn` non-empty → return it
  - else if `metadata_source` is `"bangumi"` / `"vndb"` / `"manual"` → return `name`
  - else (null / `"none"`) → return `basenameFromPath(path)`
- Export `bangumiPageUrl(id: string)` → `https://bgm.tv/subject/{id}` (alias for bangumi.tv).
- Export `vndbPageUrl(id: string)` → `https://vndb.org/{id}` (id already has the `v` prefix).
- Export `bangumiSearchUrl(query: string)` → `https://bgm.tv/subject_search/{encodeURIComponent(query)}?cat=4` (cat=4 = game).
- Export `vndbSearchUrl(query: string)` → `https://vndb.org/v?sq={encodeURIComponent(query)}`.
- Export `openExternal(url: string): void` — wraps `window.open(url, '_blank', 'noopener,noreferrer')`.

**Verify:**
- File exists, exports the 6 names listed.
- `pnpm typecheck` passes (no `any` leaks; `Pick<Game, ...>` import resolves).

**Done when:**
- `src/lib/display.ts` written with all six exports.
- typecheck green.

## Risks / pitfalls

- **`window.open` + Tauri 2 webview.** The cover_url path at line 886
  is the only existing precedent — if it turns out it doesn't actually
  open the system browser, all the new links inherit the same bug.
  Accepted: pre-existing concern, not regressed by this change. Follow-up
  if reported.
- **Bangumi `bgm.tv` vs `bangumi.tv`.** The official short host is
  `bgm.tv` (which redirects on the data-source side). Use `bgm.tv` —
  it's shorter and identical in destination.
- **VNDB search URL.** VNDB has both `vndb.org/v?sq=...` (text search)
  and `vndb.org/v/all?q=...` (faceted). The `?sq=` form is the
  canonical search box — match that.
- **`encodeURIComponent` on Chinese names.** Mandatory; both Bangumi
  and VNDB accept percent-encoded UTF-8 in the search query.
