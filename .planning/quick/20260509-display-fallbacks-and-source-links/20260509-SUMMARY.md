# Quick Task Summary: display-fallbacks-and-source-links

## What shipped

Three coordinated frontend changes plus a tiny independent UI cleanup
that landed alongside it:

1. **(1a59fdc)** `refactor(library): drop hero band — uniform density grid only`
   — pre-existing WIP in the worktree. The magazine 1.6fr+1+1+1 hero
   band ignored `--card-w` density and dramatically cropped portrait
   covers when wider than tall. Library grid is now a single
   density-driven `repeat(auto-fill, minmax(var(--card-w, 172px), 1fr))`.
   Recently-played emphasis is preserved via GameCard's status stamp.

2. **(1d54c27)** `feat(display): basename fallback + clickable BGM/VNDB links + name-search buttons`

   **Basename fallback for unmatched games.** New `src/lib/display.ts`
   exposes `displayGameName(game)`:

       name_cn (when present)
         → name (when metadata_source ∈ {bangumi, vndb, manual})
         → basenameFromPath(game.path)

   Wired into `GameCard.tsx`, `Detail.tsx`, and `MetadataPicker.tsx`
   (dialog title + initial search box seed — so the first search runs
   against what the user typed on disk).

   **Detail-page external links.** Hero BGM pill is now a clickable
   `PillLink` that opens `https://bgm.tv/subject/{id}` when bound.
   A new VNDB hero pill follows the same pattern. The right-sidebar
   `条目信息` BGM/VNDB `<dl>` rows linkify the id with `ExtAnchor`
   (small underline + ExternalLink icon).

   **搜索源 section.** New `DSection` under 路径 with two `SidebarBtn`s
   that search Bangumi (cat=4 / 游戏) and VNDB by `game.name`. The
   cleaned title is the one most likely to match upstream search; the
   basename fallback is for *display* only, not search input.

## Decisions

- **No new tauri plugin.** `openExternal()` wraps the existing
  `window.open(_, '_blank', 'noopener,noreferrer')` pattern already
  used for `cover_url` at Detail.tsx (formerly line 886, now line
  883). If Tauri 2 webview later intercepts `_blank` in-app, both the
  pre-existing cover-source link and the new ones share the bug —
  follow-up swap to `tauri-plugin-opener` would be a single-file
  change to `display.ts`.
- **Bangumi short host (`bgm.tv`)** chosen over `bangumi.tv` —
  identical destination, half the URL length.
- **VNDB search uses `?sq=`** (canonical search-box query param), not
  `/v/all?q=...` (faceted-search form).
- **Search uses `game.name` (cleaned), not `displayGameName(game)`.**
  The cleaned title strips paren-suffixes / version trails which are
  exactly what Bangumi/VNDB search filters out anyway, so it gets
  better matches than the raw basename for unmatched games.
- **MetadataPicker search seed switched to `displayGameName(game)`.**
  Different decision from the search-button helpers: the picker is the
  *bind* flow, where the basename is the user's source of truth — they
  want to confirm "this directory is this game", and the basename is
  what they recognise.
