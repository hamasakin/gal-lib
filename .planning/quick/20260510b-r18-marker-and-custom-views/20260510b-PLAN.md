# Quick Task 20260510b — R18 标识 + 自定义视图

## 范围

两个独立但同期落地的功能，共用一个 v8 migration（节省 schema 版本号）。

## A. R18/全年龄 标识 + 筛选

### 数据来源

- **Bangumi**：subject 详情有 `nsfw: bool` 字段，直接映射
- **VNDB**：现有 `tags` 查询加 `category` 字段 → 任意 `category=ero` 且 `rating>=1.5` 的 tag 视为 r18（VNDB rating 0..3，1.5 是合理阈值，避免一两个边缘 tag 误判）

### 状态空间

`age_rating: Option<String>` 三态：
- `NULL` — 未知（未抓取过 / 抓取后无信号）
- `"all_ages"` — 全年龄
- `"r18"` — R18

### Schema

`games` 表加 `age_rating TEXT NULL CHECK(age_rating IS NULL OR age_rating IN ('all_ages','r18'))`，无默认值。

### 后端改动

- `metadata/types.rs`：`MetadataDetail.is_r18: Option<bool>`
- `metadata/bangumi.rs`：`SubjectDetail.nsfw: Option<bool>`，`fetch_detail` 写 `is_r18 = nsfw`
- `metadata/vndb.rs`：`TagEntry.category: Option<String>`、查询 `fields` 加 `category`，`fetch_detail` 计算 `is_r18 = tags.any(category=ero && rating>=1.5)` 或 `None` 当 tags 为空
- `ingest.rs`：`IngestResult.is_r18: Option<bool>`（来自 `MetadataDetail` 的 `is_r18`）
- `commands.rs`：
  - `Game.age_rating: Option<String>` + `row_to_game` + `list_games`/`search_games` SELECT 加 column
  - `apply_ingest_result` 加 `age_rating = COALESCE(?, age_rating)`（保护手动覆盖；bind `is_r18.map(|b| if b {"r18"} else {"all_ages"})`）
  - `SearchFilter.age_ratings: Option<Vec<String>>`，search_games 加 WHERE `g.age_rating IN (?,...)`（支持 `null` 字符串值代表"未知"，特殊处理）
  - 新命令 `update_game_age_rating(game_id, age_rating: Option<String>)` — 详情页手动覆盖入口（白名单 `'r18'|'all_ages'|null`）
  - `lib.rs` 注册新命令

### 前端改动

- `lib/games.ts`：`Game.age_rating: "r18" | "all_ages" | null`，加 `updateGameAgeRating` wrapper
- `lib/search.ts`：`SearchFilter.age_ratings?: string[]`

  - null → 不渲染
  - 风格与现有 stamp 一致：黑底半透明 + 同色边 + mono 9px
- `routes/Detail.tsx`：更多菜单加分组 "标记年龄分级"，3 个 item：R18 / 全年龄 / 未知（清除）

## B. 自定义视图 + 批量选择

### Schema

```sql
CREATE TABLE custom_views (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE custom_view_games (
  view_id INTEGER NOT NULL REFERENCES custom_views(id) ON DELETE CASCADE,
  game_id INTEGER NOT NULL REFERENCES games(id) ON DELETE CASCADE,
  added_at TEXT NOT NULL DEFAULT (datetime('now')),
  PRIMARY KEY (view_id, game_id)
);
CREATE INDEX idx_custom_view_games_game ON custom_view_games(game_id);
```

### 后端命令（新增 6 个）

- `list_custom_views() -> Vec<CustomViewRow>` — `{ id, name, count, created_at }`，count = COUNT(custom_view_games) per view
- `create_custom_view(name) -> i64` — 返回新 view id；空白 / 重名报错
- `rename_custom_view(id, name)` — 空白 / 重名报错
- `delete_custom_view(id)` — CASCADE 自动清空 join table
- `add_games_to_view(view_id, game_ids: Vec<i64>) -> i64` — INSERT OR IGNORE，返回实际新增条数
- `remove_game_from_view(view_id, game_id)` — DELETE
- `SearchFilter.custom_view_id: Option<i64>` → WHERE `g.id IN (SELECT game_id FROM custom_view_games WHERE view_id = ?)`
- `SidebarCategories.custom_views: Vec<CustomViewRow>` — 复用 list_custom_views 的输出结构
- `lib.rs` 注册 6 个新命令

### 前端
