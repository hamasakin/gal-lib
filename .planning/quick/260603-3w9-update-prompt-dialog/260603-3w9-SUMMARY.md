# Quick 260603-3w9: 检测到新版本先提示再确认下载 Summary

把自动更新从"静默强制下载安装"改为"检测到新版本先弹 toast 提示、用户点「现在更新」才下载安装、完成后提示「立即重启」"；手动检查按钮同步走"检查→available 态显示「下载更新」→下载→重启"，不回归。

## 实现要点

- **updater.ts**：`UpdaterState` 新增 `{ phase: "available"; version; update: Update }` 分支。`checkForUpdates` 改为仅检查——命中返回 `available`（句柄塞进 state），不再 `await downloadAndInstall`；catch 仍按 silent 语义返回 idle/error。新增 `downloadAndInstallUpdate(update, opts?)`，把原下载进度逻辑（downloaded/total 累计 + emit downloading）整段搬入，完成返回 `ready`，出错返回 `error`。
- **App.tsx**：删除 `setTimeout(..., 5000)`，effect 内直接 `checkForUpdates({silent:true})`；命中 `available` 用 `toast.info` 拿 id 弹 duration:Infinity toast，action「现在更新」→ dismiss + `toast.loading` + `downloadAndInstallUpdate(state.update)` → 完成 dismiss loading，ready 弹 success「立即重启」、error 弹 error。保留 cancelled 守卫；过时注释（5s/race-with-data-dir）已重写。
- **AboutSection.tsx**：新增 `onDownload`（持 available 句柄调 `downloadAndInstallUpdate({onProgress:setState})`）；按钮区扩展 `available → 「下载更新」`、`ready → 「立即重启」`、其余 → 检查按钮；`UpdateStateLabel` 增 `available` 分支显示 `about.update_available`。
