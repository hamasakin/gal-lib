<objective>
把"自动检测更新"从"静默强制下载安装"改成"检测到新版本先提示，用户确认才下载安装"。

Purpose: 用户原话——自动检测到后不该强制安装，而是弹窗提醒"发现新版本"，让用户自己选择是否更新。重点是启动时的自动检测路径，但手动检查按钮也需保持行为一致、不回归。

Output:
- updater.ts 拆出"仅检查"与"下载安装"两个能力，新增 available 状态并把 Update 句柄暴露给调用方。

- AboutSection.tsx 手动检查改为：检查 -> 命中显示「下载更新」按钮 -> 下载 -> 「立即重启」。
- 三语 i18n 文案同步。

export type UpdaterState =

  <behavior>
    - checkForUpdates 只检查、不下载：命中返回 { phase: "available", version, update }（携带 Update 句柄），未命中返回 { phase: "up-to-date" }，出错按 silent 语义返回 idle 或 error。
    - downloadAndInstallUpdate(update, opts) 接收 Update 句柄，执行 downloadAndInstall 并通过 onProgress emit downloading 进度，完成返回 { phase: "ready", version }。
    - relaunchApp / getCurrentVersion 不变。
  </behavior>
  <action>
改造 src/lib/updater.ts，把"检查"与"下载安装"解耦（落实设计建议；保留 Tauri 的 Update 句柄，不能丢，否则无法下载）：

1. UpdaterState 联合类型新增一个 available 分支（携带 Update 句柄）：
   `| { phase: "available"; version: string; update: Update }`
   保留现有 idle/checking/up-to-date/downloading/ready/error 全部分支。

2. 改造 checkForUpdates(opts: CheckOptions) 为"仅检查"：
   - emit { phase: "checking" }。
   - `const update = await check();`
   - update 为 null -> 返回并 emit { phase: "up-to-date" }。
   - 命中 -> 返回并 emit { phase: "available", version: update.version, update }。
     （注意：把 update 句柄一起塞进 state 返回，调用方后续靠它触发下载。）
   - catch：silent -> console.warn 后返回 { phase: "idle" }；非 silent -> 返回 { phase: "error", message }。
   - 不再在此函数内调用 downloadAndInstall。

3. 新增函数 downloadAndInstallUpdate(update: Update, opts?: { onProgress?: (s: UpdaterState) => void }): Promise<UpdaterState>：
   - 把原来的下载逻辑（downloaded/total 累计 + emit { phase: "downloading", version, downloaded, total }）整段搬到这里，version 取 update.version。
   - `await update.downloadAndInstall(...)` 完成后返回并 emit { phase: "ready", version }。
   - 用 try/catch 包裹，出错返回 { phase: "error", message }（此处不需要 silent，调用方都是用户主动触发）。

4. 更新文件顶部 doc 注释，说明现在是"检查与下载分离：checkForUpdates 只探测，downloadAndInstallUpdate 由调用方在用户确认后触发"。

不要引入新依赖；继续用 @tauri-apps/plugin-updater 的 check/Update、plugin-process 的 relaunch。
