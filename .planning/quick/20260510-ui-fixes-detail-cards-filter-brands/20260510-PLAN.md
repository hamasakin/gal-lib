# Quick Task 20260510 — 4 个 UI 修复

## 问题 → 根因

### 1. 详情页人物 chip 点击跳到 `/persons/undefined`（图 4 "无效的人物 id"）

**根因（wire 协议错配）**：
- `src-tauri/src/commands.rs:2747-2755` `GameStaffRow.id: i64` 没有 `#[serde(rename = "person_id")]`
- 序列化为 `id` 字段，但 `src/lib/persons.ts:23-33` `GameStaffRow.person_id` 期望 `person_id`
- 结果：`p.person_id === undefined` → `navigate(/persons/undefined)` → `Persons.tsx:200` 显示「无效的人物 id」
- 同时 `Persons.tsx:119/147` 的 `persons.find(p => p.person_id === personId)` 永远命中不到，identity 永远是「未知人物」

### 2. FilterPanel 品牌全部 UNDEFINED（图 5）

**根因（多处 wire 错配 + 缺字段）**：
- `src-tauri/src/commands.rs:2918-2925` `FilterOptions.brands: Vec<String>` 但 `src/lib/persons.ts:65` 期望 `Array<{name, count}>` → 字符串当对象用，`it.name`/`it.count` 都是 undefined
- `src-tauri/src/commands.rs:2900-2904` `PersonOption` 没有 `count: i64` 字段，但 `src/lib/persons.ts:45-50` 期望 `count: number`
- 后端 SQL 已 SELECT `cnt`，只是没有装进 struct

### 3. FilterPanel 弹层背景透明

**根因**：`src/components/library/FilterPanel.tsx:182-185` `bg-bg-1/95 ... backdrop-blur-md` 用了半透明 + 模糊。改为纯色 `bg-bg-1`。

### 4. 侧边栏品牌列表过长 + 详情页"更多"菜单缺项

**侧边栏**：`src/components/layout/Sidebar.tsx:239-252` 把 `sidebar.brands` 全部展开，整体在外层 `ScrollArea` 滚动，没有自身限高。

**详情页"更多"按钮**：`src/routes/Detail.tsx:882-903` 当前只有「打开本地目录 / 复制路径」。卡片右键菜单（`src/components/library/GameCard.tsx:403-449`）有 7 类项：
- 启动 / 用日区启动器 / 强制结束 → 已被 LaunchButton 覆盖
- 收藏 / 取消收藏 → 已被心形按钮覆盖
- 通关状态 submenu → 已被「总览 → 常用操作」的 Select 覆盖
- **重新匹配元数据 → 缺**
- **重新抓取封面 → 缺**
