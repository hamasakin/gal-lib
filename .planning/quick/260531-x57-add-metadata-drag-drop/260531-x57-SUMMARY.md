### Task 2: 资源管理器拖拽目录添加（commit `d57857d`）

- **`src-tauri/tauri.conf.json`**：`app.windows[0]`（label `"main"`）显式加 `"dragDropEnabled": true`。v2 默认开启，显式声明防回归 + 作为契约。这是唯一的「后端侧」配置改动，未触碰任何 Rust 源码（`git diff --stat HEAD~1 HEAD -- src-tauri/src` 为空，已验证）。
- **`src/App.tsx`**：
  - 新增 `import { getCurrentWebview } from "@tauri-apps/api/webview"` + `import type { UnlistenFn }` + `addGame` / `searchGames` / `getSidebarCategories` / `useLibraryStore`。
  - 新增 `dragActive` useState 控制 overlay。
  - 新增 useEffect 监听 `getCurrentWebview().onDragDropEvent(cb)`，采用与 `useTauriListen` 同款 race-safe 清理（`cancelled` 标志 + then 内若已 cancelled 立即 fire unlisten，否则存入 `unlisten`，cleanup `unlisten?.()`）。
  - 事件分发：`enter`/`over` → `setDragActive(true)`；`leave` → false；`drop` → false + 取 `payload.paths`。
  - drop 处理：`processing` 守卫防重入；空 paths → `toast.error(dragdrop.none)`；否则**串行** `await addGame(p)` 逐个，统计 `ok` / `fail`；`ok>0` 时刷新网格+侧边栏（searchGames + getSidebarCategories 写 store），再按 `fail>0 ? toast.warning(dragdrop.partial) : toast.success(dragdrop.added)` 汇总；`ok===0` → `toast.error(dragdrop.none)`。每个失败 path `console.error` 记录。
  - drop overlay：`<main>` 加 `relative`，`dragActive` 时渲染 `absolute inset-0 z-50 pointer-events-none` 的半透明深色遮罩（`bg-black/55 backdrop-blur`，调性参照 MetadataPicker/alert-dialog overlay）+ 居中虚线框提示文字 `t("dragdrop.overlay")`。`pointer-events-none` 确保不拦截 OS 拖放。
- **i18n 三语**（zh-CN / ja-JP / en-US 同步）：新增 `dragdrop.overlay` / `dragdrop.added`（含 `{{count}}`）/ `dragdrop.partial`（含 `{{ok}}` `{{fail}}`）/ `dragdrop.none`。三语 key 数 527/527/527 对齐无空值。
