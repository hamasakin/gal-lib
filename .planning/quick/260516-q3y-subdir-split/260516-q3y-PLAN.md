<objective>
为 gal-lib 新增「整理子目录」能力：把一个被误识别成单款游戏的**品牌名父目录**条目，拆分成 N 个独立游戏条目。入口挂在 GameCard 右键菜单和 Detail「更多」下拉菜单，两边复用同一个 SubdirSplitDialog 对话框。

Purpose: 品牌父目录下通常藏着多款游戏子目录，被扫描误扫成「一款游戏」。用户需要把它拆开，让每款游戏成为可搜索、可启动、可统计的独立条目。

Output:
- 后端 2 个新 Tauri 命令 + 1 个持久化跳过表（migration 0012）
- 前端 SubdirSplitDialog 组件 + GameCard/Detail 两处入口
</objective>

   持久化跳过表 —— 拆分误扫的品牌父目录后，把父目录路径写这里，全量扫描永久跳过它（否则下次扫描重扫成重复条目）。

3. `commands.rs` 新增 `#[derive(Debug, Serialize)] pub struct SubdirEntry { name: String, path: String, clean_title: String, exe: Option<String> }`（字段 serde 默认 snake_case，前端按 snake_case 读）。

4. `commands.rs` 新增 `#[tauri::command] pub async fn list_subdirs(path: String) -> Result<Vec<SubdirEntry>, String>`：
   - 校验 `path` 是目录，否则 `Err`。
   - 用 `std::fs::read_dir` 取**直接子目录**（仅 `is_dir()`，过滤文件），对每个子目录：
     - `name` = basename
     - `path` = 子目录绝对路径字符串
     - `clean_title` = `crate::title_clean::clean_title(&name)`
     - `exe` = `crate::scan::walker::pick_best_exe(&child_path).map(|p| p.to_string_lossy().into_owned())`
   - 按 `name` 排序后返回。read_dir 单项 error 用 `filter_map(Result::ok)` 忽略（沿用 walker.rs 风格）。

5. `commands.rs` 新增 `#[tauri::command] pub async fn split_game_into_subdirs(game_id: i64, paths: Vec<String>, state: State<'_, AppPaths>) -> Result<Vec<i64>, String>`：
   - `let pool = state.pool().await.map_err(err_str)?; let data_dir = state.data_dir.clone();`
   - 先 `SELECT path FROM games WHERE id = ?` 取原父目录路径（找不到则 `Err("游戏不存在")`）。
   - 对 `paths` 每个 path：构造 `DiscoveredGame { path: PathBuf::from(&p), raw_name: <basename>, clean_name: clean_title(basename), executable: pick_best_exe(&path) }`，调 `ingest_one_dir(&*pool, &data_dir, &dg).await?` —— 这一步即建条目 + 走元数据匹配。收集返回的 `id` 进 `Vec<i64>`。空 `paths` 直接 `Err("未选择任何子目录")`。
   - 全部成功后：`DELETE FROM games WHERE id = ?` 删原父目录条目（scan_review_queue 有 ON DELETE CASCADE，无需手动清；若无 CASCADE 则先 `DELETE FROM scan_review_queue WHERE game_id = ?`，按现有 schema 决定）。
   - `INSERT OR IGNORE INTO scan_skip_dirs (path) VALUES (?)` 写入原父目录路径。
   - 返回新建 id 列表。
   - 注意：边界情况 —— 若某个 path 等于原父目录路径本身则跳过（防自包含）；若 path 已是某个 `games.path`（INSERT 走 ON CONFLICT 更新），`ingest_one_dir` 内 `insert_placeholder_dir` 用 `ON CONFLICT(path)` 已幂等，无需特判。

6. `commands.rs` 修改 `start_scan`：在构造 `existing_paths` 的 HashSet 后，追加查询 `SELECT path FROM scan_skip_dirs` 并把结果 `PathBuf` 也 `insert` 进 `existing_paths`（同一个跳过集，run_scan 会一并跳过）。在该处加简短注释说明 scan_skip_dirs 来源。

7. `lib.rs`：`invoke_handler` 的 `generate_handler!` 列表末尾追加 `commands::list_subdirs, commands::split_game_into_subdirs,`，并加一行注释（如 `// Quick 260516-q3y — subdir split`）。
