# Quick 20260512e — 双滚动条修复 + Sidebar 重构

## 背景

用户在使用图书馆时反馈两个 UI 问题：

1. **最外层又多了一层滚动条**：浏览器主滚动条 + Library 内部滚动容器并存，出现双滚动条。
2. **侧边栏太乱**：当前 6 组 Section（视图 / 通关状态 / 工具 / 我的视图 / 自定义标签 / 品牌·厂牌 / 发行年份）排序不合理，且「品牌」「发行年份」与 toolbar 上 `FilterPanel` 多维筛选完全重叠 — 是 v1.0 单字段筛选的遗留。

## 诊断

### 滚动条根因

`src/App.tsx:54` 的 `<main>` 仅有 `min-w-0 flex-1`，**缺 `min-h-0` 和 `overflow-hidden`**。当 `<Outlet>` 渲染 Library 时，Library 根 `flex h-full w-full flex-col` 的 `h-full` 解算依赖父容器的高度约束 — 没有 `min-h-0`，子内容（Toolbar/Header/ScanProgressBar/BackfillProgressBar/ActiveSessionBar）累计高度撑爆 main，触发外层（即 `<div className="flex min-h-0 flex-1">` 的滚动），同时 Library 内部 `scrollContainerRef`（`overflow-auto`）也滚 — 这就是用户看到的双层。

### 侧边栏重叠

`FilterPanel` (`src/components/library/FilterPanel.tsx`) 已经提供：
- 品牌（带搜索，line 332-350）
- 发行年份（chip 多选，line 259-285）
- 编剧 / 画师 / 声优 / 官方标签

而 `Sidebar` 仍渲染：
- 品牌·厂牌（line 469-490，写死 max-h 300px 内嵌滚动）
- 发行年份（line 492-505）

两处入口分别走 `filter.brand` / `filter.year_decade`（旧单字段）和 `advFilter.brands` / `advFilter.years`（新多维），用户在 toolbar 加了筛选 chip 后看不到侧边栏被点亮，体验混乱。

## 修改

### A. 修复双滚动条 — `src/App.tsx`

`<main>` 类名从：
```
min-w-0 flex-1 bg-background
```
改为：
```
flex min-h-0 min-w-0 flex-1 flex-col overflow-hidden bg-background
```

- `min-h-0` 让 flex 子项（Library 根）的 `h-full` 严格约束在 main 自己的高度内
- `overflow-hidden` 把滚动责任全部交给 Library 内部 `scrollContainerRef`（已经写好的 `overflow-auto`）
- `flex flex-col` 让 Outlet 子元素正确撑满

### B. 重设计 Sidebar — `src/components/layout/Sidebar.tsx`

**新结构（自上而下）：**

```
─ ScrollArea ────────────────
图书馆 (Section)
  · 图书馆全景
  · 收藏夹
  · 扫描复核（含 review_pending 红点 badge）

通关状态 (Section)
  · 游玩中 / 已通关 / 未开始 / 弃坑

我的视图 (Section + ＋)
  · custom views…

自定义标签 (Section，仅当存在)
  · # tag…
─────────────────────────────

─ 底部固定区 (border-t)
  · 游玩统计
  · 截图集
  · 设置
```

**变更摘要：**

1. **删除"工具"Section** — 其下 4 行（扫描复核/游玩统计/截图集/设置）按职能拆分：
   - **扫描复核** 归到"图书馆"组（属于 library 视图，需配合 review_pending 红点常驻 sidebar 主区）
   - **游玩统计 / 截图集 / 设置** 移到底部固定区（次要导航，不参与筛选）
2. **删除"品牌 · 厂牌"组** — 与 FilterPanel 重复。一并删除 `isBrandActive` 函数和 `sidebar.brands` 渲染。
3. **删除"发行年份"组** — 同上，一并删除 `isDecadeActive`。
4. **首个 Section 标签重命名** — "视图" → "图书馆"，更精确反映"图书馆全景 + 收藏夹 + 扫描复核"这一组的含义；"我的视图"已是单独 Section。
5. **顺序优化** — 图书馆 → 通关状态 → 我的视图 → 自定义标签，从"全局入口"到"用户定义"自然递进。

**保留：**
- `filter.brand` / `filter.year_decade` 后端字段（`src/lib/search.ts` SearchFilter）不动 — 仍可被未来代码设置；本任务只移除 UI 入口。
- `FilterChip.tsx` 不动 — brand/year_decade chip 逻辑保留作为兜底显示。
- icon-mode（窄边栏）仍然可用。
