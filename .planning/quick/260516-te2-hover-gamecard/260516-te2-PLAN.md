```ts
// 已有：右键菜单的「收藏」toggle 走的就是这个函数。本计划的新按钮复用它，不要新写。
async function onToggleFavorite() {
  const next = !game.is_favorite;
  try {
    await updateGameFavorite(game.id, next);
    onMutated?.();
    toast.success(next ? "已收藏" : "已取消收藏");
  } catch (err: unknown) {
    toast.error(`操作失败 — ${String(err)}`);
  }
}
```

目标交互：
1. 新增一个 hover 时出现的快捷收藏按钮（圆形 `<button>`）。
   - 默认（非 hover）`opacity-0`，`group-hover:opacity-100`（卡片根 div 已有 `group` class），加 `transition-opacity duration-200` 与现有 hover overlay 一致。
   - 容器需 `pointer-events-auto`（封面里 z-[2] 的 hover overlay 是 pointer-events-none，但本按钮在独立的 z-[3] 容器里，确保按钮自身可点击）。
2. 按钮内图标用已 import 的 lucide-react `Heart`：
   - `game.is_favorite === true` → `<Heart fill="currentColor" />`（实心，按钮底色高亮，复用 `text-brand` / brand 配色，表示「已收藏，点此取消」）。
   - `game.is_favorite === false` → `<Heart>` 空心描边（按钮底色用半透明深色 `bg-black/55 backdrop-blur` + 描边，表示「点此收藏」）。
   - 图标 `size` 约 14；按钮尺寸约 `h-[28px] w-[28px]`，`rounded-full`，`grid place-items-center`，加 `shadow-lift transition-transform hover:scale-110`，对齐既有启动按钮风格但略小。
3. 按钮 `onClick`：先 `e.stopPropagation()`，再 `void onToggleFavorite()`（复用既有函数，不要新写收藏逻辑、不要新增 IPC 调用）。
4. 无障碍：`type="button"`、`aria-label` 与 `title` 按状态切换为「收藏」/「取消收藏」。
5. 既有的「非 hover 时已收藏游戏显示的实心 heart 标记」要保留 —— 它是静态收藏指示器（远观即知是否已收藏）。处理方式：把静态 heart 标记包一层 `group-hover:opacity-0`（hover 时淡出），让 hover 时由快捷按钮接管这个角落，避免静态 heart 与按钮重叠堆叠。未收藏游戏静态标记本就不渲染，hover 时只出现按钮 —— 行为正确。
6. 选择模式抑制：整个快捷收藏按钮在 `selectMode === true` 时不渲染（与既有 stamp / 启动 overlay 在 select 模式下的处理一致）。静态 heart 标记是否在 select 模式显示维持现状即可，不在本任务范围内改动。

不要改动：onToggleFavorite 函数体、右键菜单的「收藏」项、GameGrid.tsx、任何后端 / IPC / DB 文件。本任务是单文件、纯 JSX/Tailwind 改动。
