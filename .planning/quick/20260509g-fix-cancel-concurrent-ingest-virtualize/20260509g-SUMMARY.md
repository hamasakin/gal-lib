# Quick Task 20260509g — 修扫描取消 + 跨 game 并发 ingest + 跨 game query dedup + GameGrid 重启虚拟化

## One-liner

ScanContext.cancel 检查点补到 start_scan 占位 INSERT / ingest 循环 / refresh_all_metadata 循环；start_scan ingest 改为 JoinSet INGEST_CONCURRENCY=4 并发 + 跨 game OnceCell QueryCache 防 thundering-herd；GameGrid 用 useVirtualizer row-mode + ResizeObserver-driven columnCount 重启虚拟化（scrollContainer 复用 Library.tsx 的 flex-1 overflow-auto wrapper）。

## Tasks Executed

### Task 1 — 修扫描取消（cancel checkpoint 补全）

**Commit:** `de4ac19` — `fix(quick-20260509g): add cancel checks to scan + refresh_all_metadata loops`

**Changes (`src-tauri/src/commands.rs`):**
- start_scan 占位 INSERT 循环（`for dg in &discovered`）顶部加 `if ctx.cancel.load(Relaxed)` → emit `ScanStatus::Cancelled` → return
- start_scan ingest 阶段（已经迁移到 JoinSet 在 Task 2，cancel check 落在 spawn 闭包顶部，外层 spawn-loop 也在 spawn 前先看 cancel flag）
- refresh_all_metadata 改造：新签名加 `scan_state: State<'_, ScanState>`；进入 spawn 前 `*scan_state.ctx.lock() = Some(Arc::new(ScanContext::new()))`（仿 start_scan 240-245 模式），spawn 闭包内循环顶部检查 `ctx_for_task.cancel.load(Relaxed)` → emit Cancelled with `completed: i` → return

**Result:** 取消按钮（cancel_scan 命令）现在能停 3 条 path：
- start_scan 占位 INSERT 阶段
- start_scan ingest 阶段（in-flight 网络请求完成后下一 task 顶部 cancel check 让它早退）
- refresh_all_metadata（跟 start_scan 共享 ScanState，cancel_scan 一并影响）

### Task 2 — 跨 game ingest 并发（INGEST_CONCURRENCY=4） + 跨 game query dedup cache

**Commit:** `5c1d830` — `feat(quick-20260509g): cross-game ingest concurrency + query dedup cache`

**Changes (`src-tauri/src/ingest.rs`):**
- 新增 `pub type QueryCache = Mutex<HashMap<String, Arc<OnceCell<Option<Candidate>>>>>` — scope 限定为单次 start_scan 调用
- 新增 `pub fn new_query_cache() -> Arc<QueryCache>`
- 新增 `pub async fn pick_best_with_cache(query, &cache) -> Option<Candidate>` — Mutex 短锁取/插 OnceCell，再 OnceCell::get_or_init 防 thundering-herd；首 caller fetch，余 caller await 同一 future；返回 Clone 给 each caller 独立 owned 值
- 新增 `pub async fn process_game_cached(...)` — process_game 主体复刻，把两处 `pick_best_across_sources` 替换为 `pick_best_with_cache`（标准 query + aggressive_candidates fan-out 都走 cache）

**Changes (`src-tauri/src/commands.rs`):**
- 新常量 `const INGEST_CONCURRENCY: usize = 4`
- 抽出 `apply_ingest_result(pool, game_id, &res)` helper — `enrich_metadata_for_dir` 与 start_scan 并发路径共用 SQL（去重）
- start_scan ingest 循环改造：`for (i, dg) in discovered.into_iter().enumerate()` → `JoinSet<()>` + 主循环 `while set.len() < INGEST_CONCURRENCY { spawn... } set.join_next().await`；cancel 检查在 spawn 前 + spawn 闭包顶部 + 占位 INSERT 之后
- 跨 game completed counter 改为 `Arc<AtomicUsize>`，`fetch_add(1, Relaxed) + 1` emit progress（并发后顺序乱，单调递增正确）
- 终态 emit：cancel flag flipped → `ScanStatus::Cancelled` with `completed.load()`；否则 → `ScanStatus::Completed` with `completed: total`

**保持现状（不动）：**
- `pick_best_across_sources`（同 game Bangumi+VNDB tokio::join! 并发，v1.0 设计）
- `process_game`（add_game / refresh_metadata / refresh_all_metadata / bind_metadata 走的还是它，无 cache）
- `aggressive_candidates` fan-out 在 process_game / process_game_cached 都是 sequential
- governor token-bucket 限速器（已 process-wide thread-safe）

**Result:** 116-game 扫描时 ingest 阶段 ~3x 提速（4 个 task 并发卡在 governor 限速器，不会突破 Bangumi 1 req/s + VNDB 100 req/min）；同一 cleaned query 跨 game 只发一次 Bangumi+VNDB 搜索。

### Task 3 — GameGrid 重启 react-virtual 行虚拟化

**Commit:** `fa2b7aa` — `feat(quick-20260509g): GameGrid 重启 react-virtual 行虚拟化`

**Changes (`src/components/library/GameGrid.tsx`):**
- 完整重写，import `useVirtualizer` from `@tanstack/react-virtual` (^3.13.24，已在 deps)
- Props 加 `scrollContainerRef: React.RefObject<HTMLDivElement | null>`（React 19 写法，显式 `| null`）
- 移除自身的 `<div className="h-full w-full overflow-auto">` 外层；GameGrid 不再自己套 overflow-auto
- ResizeObserver 监听内层 `<div ref={innerRef} className="px-8 pb-20 pt-7">`：
  - `getComputedStyle(el).getPropertyValue("--card-w")` 读 CSS var → setCardWidth
  - `cols = max(1, floor((clientWidth - 64 + COLUMN_GAP) / (cardW + COLUMN_GAP)))` → setColumnCount（auto-fill 等价公式）
- `useVirtualizer({ count: ceil(games.length / columnCount), getScrollElement: () => scrollContainerRef.current, estimateSize: () => rowStride, overscan: 5 })`
- rowStride = cardWidth*(4/3) + 56 (META_HEIGHT) + 28 (ROW_GAP)
- 当 rowStride / columnCount 变化时主动 `virtualizer.measure()` 让 internal size cache 失效（保证 DensityToggle 切换 / viewport resize 后 row 位置正确）
- 渲染：`<div style={{ height: totalSize, position: relative }}>` 内部 `getVirtualItems().map(row => <div style={{ position: absolute, transform: translateY(row.start), display: grid, gridTemplateColumns: repeat(columnCount, minmax(0, 1fr)), columnGap: 22, rowGap: 28 }}>...{rowGames.map(GameCard)}</div>)`

**Changes (`src/routes/Library.tsx`):**
- 加 `const scrollContainerRef = useRef<HTMLDivElement>(null)`（useRef 已在 line 21 import）
- line 258 的 `<div className="flex min-h-0 flex-1 flex-col overflow-auto">` 加 `ref={scrollContainerRef}`
- 调用 GameGrid 时传 `scrollContainerRef={scrollContainerRef}`

**Result:** 116+ game 库滚动 DOM 中只渲染 (overscan*2 + visible) 行 × columnCount 个 GameCard（典型 viewport ~5 rows × 6 cols = 30 卡 + overscan ≈ 80 卡，不再是 116+ 全量 plain DOM）；DensityToggle 切换 `--card-w` 触发 ResizeObserver → measure → columnCount 重算 → virtualizer reflow。
