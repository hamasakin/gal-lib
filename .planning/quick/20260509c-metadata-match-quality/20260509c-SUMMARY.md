## Truths delivered (against PLAN.md must_haves)

- ✅ `Candidate.confidence` is the **max** of normalized similarity scores across `(title, name_cn, alias[])` — `match_score::score_best`, used by both `bangumi.rs` and `vndb.rs`. Verified by new unit test `score_best_takes_max_across_candidates`.
- ✅ `ingest::process_game` searches Bangumi AND VNDB **in parallel via `tokio::join!`**, merges into one `Vec<Candidate>`, and picks the highest-scoring `≥80` candidate regardless of source.
- ✅ Bangumi search **no longer truncates to top-5** — full result list scored.
- ✅ `title_clean` strips `[...]` / `【...】` symmetrically with `(...)`, plus trailing `+ Tokuten Voice` / `+ ボーカル特典` / `DISC-N`.
- ✅ When standard clean misses both sources, `aggressive_clean` (longest contiguous CJK run) retries once. Implemented in `ingest.rs::process_game`.
- ✅ `GameCard` renders `「待复核」` in **bottom-left**, not top-right. Bottom-left precedence is `获取中 > 待复核 > 无 EXE` (encoded as `bottomBadge` constant).

## Known minor edge cases (not blocking)

Spotted during the F:\galgame walk; defer unless they show up as misses
in real-world scan:

1. **Trailing `+ Update 1.01` / `+ アップデートパッチ Ver.1.01`** survives
   `RE_TOKUTEN` because the regex char class doesn't include digits/dots.
   The simpler `+ Tokuten Voice` (the user's example case) works.
   Future fix: add `RE_VERSION_SUFFIX = r"\s*\+\s*(?:Update|Ver\.?)\s*[\d.]+\s*$"`.
2. **`(18禁ゲーム) [240830] ... ドゥアズ～神父様ノ言いなり～`** — STD clean
   keeps the title intact (`ドゥアズ～神父様ノ言いなり～`), so this is fine
   in practice; the aggressive fallback only fires when STD misses both
   APIs, and even then it correctly isolates a CJK substring.
3. **`(1)` suffix on `.rar (1)` filenames** — not stripped by
   `strip_archive_ext` in the example tool. Not a real-world issue (a
   real game directory wouldn't have this; only the example file).
