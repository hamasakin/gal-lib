# Quick Task: metadata-match-quality

## Goal

Three coordinated improvements to lift auto-bind hit rate and clarify
the "needs review" affordance:

1. **Match scoring covers all candidate names** — Bangumi `name_cn` and
   VNDB `titles[]` were collected but never scored against, so a CJK
   directory name vs. a romaji canonical title scored 0 even when the
   `name_cn` was a perfect match. Score against the candidate's full
   title pool, take the max.

2. **Parallel sources, merged pick** — current `Bangumi-first, VNDB
   fallback only on miss` lets a wrong Bangumi >=80 hit suppress a
   correct VNDB hit. Search both, merge, pick best ≥80.

3. **Cleaner cleaning + cascade** — `[PULLTOP] [180216]`-style
   release-group brackets, `+ Tokuten Voice` / `DISC-15` suffixes,
   `(iso+mds+rr3)` extension blobs survive the current 5-step pipeline
   and contaminate the search query. Add bracket stripping + suffix
   regexes; on a double miss, retry with an aggressive clean (longest
   contiguous CJK run).

4. **「待复核」 affordance moves to bottom-left** — replaces the right-top
   corner stamp; signals a TODO action, sits next to other actionable
   badges (no-exe).

## Non-goals

- No change to the AUTO_BIND_THRESHOLD (stays 80).
- No new metadata source (DLsite, Getchu, etc.).
- No change to manual MetadataPicker UI — already presents top-N from
  both sources.
- No DB migration: existing rows keep their bound source untouched
  unless the user re-runs a refresh.

## Tasks

### Task 1 — Multi-name confidence + parallel ingest

**Files:**
- src-tauri/src/metadata/match_score.rs (add `score_best`)
- src-tauri/src/metadata/bangumi.rs (use `score_best`; drop `take(5)`)
- src-tauri/src/metadata/vndb.rs (use `score_best` over title + titles[])
- src-tauri/src/ingest.rs (replace fallback with `tokio::join!` + merge)

**Action:**
1. In match_score, add `pub fn score_best(query: &str, candidates: &[&str]) -> u8` that returns the max of `score(query, c)` across the slice. Keep `score` for callers that have a single string.
2. In bangumi.rs::search, replace the per-hit `score(&query, &s.name)` with `score_best(&query, &[&s.name, s.name_cn.as_deref().unwrap_or("")])`. Drop the `.take(5)` so all hits participate.
3. In vndb.rs::search, build the title pool from `hit.title` + every `hit.titles[].title` and call `score_best`.
4. In ingest.rs::process_game, replace the sequential `bangumi → fallback vndb` with `tokio::join!(bangumi::search, vndb::search)`, merge the two `Vec<Candidate>`s, sort by confidence desc, pick the first ≥80.

**Verify:**
- `cargo test -p gal-lib --lib metadata::match_score` (existing 5 tests pass + new `score_best_takes_max` test)
- `cargo build` clean
- Behavior smoke: query "CLANNAD" against a hand-built candidate where `name="CLANNAD - 全年齢版"` and `name_cn="CLANNAD"` should yield 100 (was ~70).

**Done:** When a Bangumi hit's `name` is romaji-distant but `name_cn` is exact, the candidate scores 100. When both Bangumi (low conf) and VNDB (high conf) return hits, VNDB wins.

### Task 2 — Title clean: brackets + tokuten + cascade fallback

**Files:**
- src-tauri/src/title_clean.rs (add `RE_BRACKET`, `RE_TOKUTEN`, `RE_DISC`, `aggressive_clean`)
- src-tauri/src/ingest.rs (call `aggressive_clean` on double-miss)

**Action:**
1. Add `RE_BRACKET = r"[\[【][^\[\]【】]*[\]】]"` to the pipeline (between paren-strip and noise-strip).
2. Add `RE_TOKUTEN = r"\s*\+\s*[\p{Han}\p{Katakana}\p{Hiragana}A-Za-z]+(\s*Voice|\s*特典)?\s*$"` to peel `+ Tokuten Voice` / `+ ボーカル特典`.
3. Add `RE_DISC = r"\s*DISC[-_\s]*\d+\s*"` to strip `DISC-15`.
4. Add `pub fn aggressive_clean(raw: &str) -> String` that returns the **longest contiguous run** of `\p{Han}|\p{Katakana}|\p{Hiragana}` characters in the cleaned string. Falls back to the standard clean if no CJK run exists.
5. In ingest::process_game, after the parallel-merge in Task 1: if the best candidate is `<80`, run `aggressive_clean(&discovered.raw_name)`, retry both sources once, merge, pick again.

**Verify:**
- `cargo test -p gal-lib --lib title_clean` (existing 6 tests pass + 3 new: `strips_brackets`, `strips_tokuten_suffix`, `aggressive_picks_longest_cjk`)
- Worked example: `"(18禁ゲーム) [PULLTOP] [180216] PULLTOP 15th X 15タイトル Premium Anniversary Pack DISC-15 見上げてごらん、夜空の星を FINE DAYS (iso+mds+rr3)"` → standard clean strips brackets+disc, aggressive_clean isolates `見上げてごらん、夜空の星を` (or the longest CJK run).

**Done:** When standard clean returns a noisy 80-char query that misses both APIs, the aggressive retry isolates the core title and finds a hit.
