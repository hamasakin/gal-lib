# Quick 20260512g — 修复"选视图后页面不变"

## 现象

侧边栏点击"我的视图"下的某条自定义视图，左侧 active 高亮正常切换，但
主区列表仍展示全库；多选 age_ratings 时同样不生效。

## 根因

`src/routes/Library.tsx` 顶部的 `isFilterEmpty(f: SearchFilter)` 漏列了
两个字段：`custom_view_id` 和 `age_ratings`。

流程：
1. Sidebar 点击 → `applyFilter({ custom_view_id: id })` → store `filter = { custom_view_id: id }`
2. `refetchGrid` 把 store filter 和 advFilter 合并成 `merged`
3. `isFilterEmpty(merged)` 由于不看 `custom_view_id`，认定为"空"
4. 于是 `searchGames(query, sort, null)` —— null filter 传到后端
5. 后端无视 custom_view_id，返回全库；前端列表与 sidebar active 不一致

后端 `src-tauri/src/commands.rs:2043` 实际上是支持 `custom_view_id` 过滤的，
所以这是纯前端的 bug。

## 修改

**`src/routes/Library.tsx:83-94`** — 在 `isFilterEmpty` 中追加：

```ts
function isFilterEmpty(f: SearchFilter): boolean {
  return (
    f.tag_id == null &&
    f.status == null &&
    !f.favorite &&
    f.brand == null &&
    f.year_decade == null &&
    f.custom_view_id == null &&                                    // ← 新增
    (f.brands == null || f.brands.length === 0) &&
    (f.staff_ids == null || f.staff_ids.length === 0) &&
    (f.official_tags == null || f.official_tags.length === 0) &&
    (f.age_ratings == null || f.age_ratings.length === 0)          // ← 新增
  );
}
```
