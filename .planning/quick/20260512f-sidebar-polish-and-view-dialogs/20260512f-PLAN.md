# Quick 20260512f — Sidebar 打磨 + 视图对话框

## 背景

用户反馈 20260512e 的成果"视图侧边栏丑"且"新建视图 Modal 不认真"。前一次 quick 只搬动了结构，没碰视觉细节，也保留了三处 `window.prompt` / `window.confirm` —— 跟应用的"图书馆/胶片"印章风完全不搭。

## 修改

### A. 新建对话框组件

**`src/components/library/ViewNameDialog.tsx`**（新）— 复用对话框，承载新建 / 重命名两种模式：
- shadcn `Dialog` 包装，sm 宽度 400px，sharp 圆角 `var(--r-md)`
- Header：8×8 印章方块 + serif 标题 + mono uppercase 副标
- Body：mono 小标签 + 卡片化 input + 字符计数 (60 上限提示)
- Footer：分割线 + bg-bg-0，副按钮 outline / 主按钮 brand
- 控制反转：父持有 mode/onClose/onSubmit；onSubmit 可返回 Promise，busy 期间锁定关闭
- 行为：autofocus + select，回车提交，Esc 取消，超长 / 未改动 / 空时禁用主按钮

**`src/components/library/DeleteViewDialog.tsx`**（新）— 删除确认：
- shadcn `AlertDialog`，size sm
- 印章红 glyph + serif 标题 + mono 副标
- 高亮展示视图名 + 计数（bg-2 卡片块）
- 红色 destructive 主操作（`#d96f5a` 印章红），副操作 outline
- busy 期间禁用所有按钮，防止双触发

### B. Sidebar 视觉打磨 — `src/components/layout/Sidebar.tsx`

**SectionLabel 改进：**
- 加左缘 6px 印章红短线（cinema clapboard 标记感）
- mono uppercase 9.5px tracking 0.16em（原 0.14em，更舒展）
- 顶部 mt-3 / mb-1，section 之间有真正的呼吸
- 支持 `action` slot — "我的视图" 的 ＋ 按钮与标题同行 baseline 对齐

**SidebarRow 改进：**
- 固定 32px 高度，所有图标 size=14 strokeWidth=1.6 统一
- active 态：bg-brand-soft + 2px brand 左缘实线（绝对定位伪元素）+ 图标变 brand 色
- hover 态：图标 ink-2 → ink-1，数字 ink-3 → ink-2（group hover 联动）
- 数字字号 10.5px tabular-nums mono，与文字基线对齐

**CustomViewRow 新组件：**
- hover 时数字淡出、`MoreHorizontal` (⋯) DropdownMenu 触发器淡入（重命名 / 删除）
- DropdownMenu 打开时持续显示（data-state=open）
- 不依赖右键菜单（普通用户发现不了），右键作用废弃

**空态：**
- "还没有视图" 单行文案 → dashed border 卡片 CTA，hover 时变 brand 边框 + brand-soft 底
- 文案 "＋ 新建第一个视图"

### C. 入口归一 — `src/routes/Library.tsx`

批量选择面板里的 `window.prompt("新视图名称")` 也替换为 `ViewNameDialog`，与侧边栏共用一个组件，UX 一致。

### D. 删除的代码

- `Sidebar.tsx` 的 `ContextMenu` import + JSX（被 DropdownMenu 替代）
- `Sidebar.tsx` 的 `renamingViewId` / `renameDraft` 内联重命名 input（被 Dialog 替代，并解决其点击外部触发 onBlur 误提交的问题）
- 三处 `window.prompt` / `window.confirm` 调用
