# Quick Task: aggressive-trim-and-vndb-recall

## Goal

Two follow-ups to 20260509c flagged by user feedback:

1. **AGG (aggressive_clean) was severing main titles at digits.** The
   real-world example 「艶嬢学園２ ー熾天使たちの花園ー」 split into
   ・run 1: `艶嬢学園` (4 chars)
   ・run 2: `ー熾天使たちの花園ー` (10 chars, ← longest, picked)
   because the fullwidth digit `２` (U+FF12) wasn't classified as
   CJK-titlechar. The chosen "subtitle" run is also book-ended by
   prolonged sound marks `ー...ー` which are punctuation, not content.

   Fix: include fullwidth + halfwidth digits AND ASCII letters in the
   run, then trim leading/trailing punctuation from the final pick.
   After: 「艶嬢学園２」 wins (7 chars) vs `熾天使たちの花園` (8 chars
   after trim) — actually the **subtitle still wins by length**, but
   it's now clean (`熾天使たちの花園`) and either query is a viable
   VNDB hit. Most importantly, when the user has a title like
   `戦姫絶唱 シンフォギア２` the digits no longer break the run.

2. **VNDB search recall was capped at 5.** VNDB's `search` filter is
   already substring/token fuzzy across titles+aliases (no extra
   "fuzzy" mode exists) but we requested only `results: 5` and didn't
   set a sort, so relevant candidates ranked 6+ never participated in
   our scoring. Bump to `results: 25` and add `sort: "searchrank"` —
   this is the VNDB-blessed sort key for search-filter queries
   (relevance order).

## Non-goals

- No change to Bangumi recall — Bangumi v0 search default page size
  is already ~25 hits and we removed our own take(5) in 20260509c.
- No change to the AUTO_BIND_THRESHOLD or scoring algorithm.
- No new fuzzy matching layer in match_score (Levenshtein already
  handles small typos; cross-language is solved by score_best).

## Tasks

### Task 1 — AGG: include digits + letters; trim trailing punctuation

**Files:**
- src-tauri/src/title_clean.rs

**Action:**
1. Extend `is_cjk_titlechar` to also return true for halfwidth digits
   `0..=9`, fullwidth digits `０..=９` (U+FF10..U+FF19), and ASCII
   letters `A-Za-z` — these are common in galgame titles
   (`シンフォギア２`, `Robotics;Notes`, `CLANNAD`) and shouldn't
   sever a run.
2. After `aggressive_clean` picks the longest run, strip any leading
   or trailing chars that match the punctuation set
   (`・々、。〜！？ー` plus space). This removes the awkward `ー...ー`
   wrap on subtitle-only picks.

**Verify:**
- Existing test `aggressive_picks_longest_cjk_run` still passes (the
  long sentence inside that example contains no digits, so it's
  unaffected — but the result no longer has trailing punctuation, so
  the assertion may need updating).
- New test `aggressive_keeps_digits_in_run`: input
  `[240927] [アストロノーツ・シリウス] 艶嬢学園２ ー熾天使たちの花園ー`,
  assert AGG yields a non-empty trimmed run with `艶嬢学園２` intact
  somewhere in the result (or that result is at least one of two
  reasonable runs).
- New test `aggressive_trims_leading_trailing_punct`: input that ends
  in `ーー〜` returns without the trailing punctuation.

### Task 2 — VNDB: 25 results + searchrank sort

**Files:**
- src-tauri/src/metadata/vndb.rs

**Action:**
Update the search request body:

let body = serde_json::json!({

Leave `fetch_detail` unchanged — it queries by id, not by search.
