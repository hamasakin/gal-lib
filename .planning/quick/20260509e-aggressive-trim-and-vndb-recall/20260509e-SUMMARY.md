## VNDB note

The user asked whether VNDB supports fuzzy search. Researched: VNDB's
`search` filter on `/kana/vn` is **already** a fuzzy substring/token
match across visible titles + aliases — there is no extra "fuzzy"
mode to enable. What we were missing was page size. The default 5 we
used (vs the API default 10, max 100) was capping recall on common
titles where the canonical entry ranks 6–15 behind alternate releases,
fan-discs, and spinoffs. 25 is wide enough to catch off-by-name
candidates, still cheap on the 100req/min limiter. `sort:
"searchrank"` is VNDB's relevance order for search-filter queries.
